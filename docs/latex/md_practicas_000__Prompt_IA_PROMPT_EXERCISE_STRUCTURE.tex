The main specification file that previously would be named {\ttfamily README.\+md} MUST instead be named using the following pattern\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{<learning\_mode>-\/-\/<difficulty>.md}

\end{DoxyCode}


Where\+:
\begin{DoxyItemize}
\item {\ttfamily \texorpdfstring{$<$}{<}learning\+\_\+mode\texorpdfstring{$>$}{>}} is the exact value of the {\ttfamily learning\+\_\+mode} field in the metadata block (e.\+g., {\ttfamily concept}, {\ttfamily guided}, {\ttfamily project}, {\ttfamily challenge}).
\item {\ttfamily \texorpdfstring{$<$}{<}difficulty\texorpdfstring{$>$}{>}} is the exact value of the {\ttfamily difficulty} field in the metadata block (e.\+g., {\ttfamily beginner}, {\ttfamily easy}, {\ttfamily medium}, {\ttfamily hard}, {\ttfamily challenge}).
\end{DoxyItemize}

Rules\+:
\begin{DoxyItemize}
\item Use all lowercase.
\item Separate the two parts with exactly two hyphens ({\ttfamily -\/-\/}).
\item No spaces or additional suffixes/prefixes.
\item The file must reside inside the folder corresponding to its topic, e.\+g.\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{Practicas/}
\DoxyCodeLine{  3. Variables/}
\DoxyCodeLine{    my-\/variables-\/exercise/   \# Folder name is flexible but descriptive}
\DoxyCodeLine{      concept-\/-\/beginner.md   \# Specification file (this template applied)}
\DoxyCodeLine{      src/}
\DoxyCodeLine{      respuestas/}

\end{DoxyCode}

\item One exercise = one folder containing exactly one specification file following this convention.
\item If multiple exercises share the same topic, each still has its own folder and one spec file named by this convention.
\end{DoxyItemize}

Examples\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{concept-\/-\/beginner.md}
\DoxyCodeLine{guided-\/-\/medium.md}
\DoxyCodeLine{project-\/-\/hard.md}
\DoxyCodeLine{challenge-\/-\/challenge.md}

\end{DoxyCode}


Rationale\+: This naming scheme allows automated tooling to classify exercises by pedagogical mode and difficulty without parsing file contents.
